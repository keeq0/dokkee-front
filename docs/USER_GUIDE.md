# Руководство пользователя Dokkee

Dokkee — веб-сервис для проверки и анализа документов с помощью искусственного интеллекта. Сервис принимает документы пользователя, выделяет в тексте потенциальные риски (юридические, финансовые, признаки мошенничества), формирует рекомендации и позволяет уточнять детали в чате с ассистентом.

## 1. Назначение и состав сервиса

Сервис Dokkee состоит из двух приложений:

- **Фронтенд** — веб-интерфейс на Vue 3, открывается в браузере.
- **Бэкенд** — серверное приложение на Go, обрабатывает аутентификацию, хранит документы и результаты анализа, обращается к ИИ-модели.

Пользователь работает только с фронтендом. Бэкенд скрыт за HTTP API и доступен по адресу, который задаёт администратор при развёртывании.

Поддерживаемые форматы документов: **PDF**, **DOC**, **DOCX**. Ограничение по размеру одного файла — 10 МБ.

## 2. Регистрация и вход

При первом обращении к сервису необходимо создать учётную запись. Форма регистрации запрашивает:

- логин (уникальный, не менее 3 символов);
- пароль;
- фамилию, имя, отчество;
- электронную почту;
- номер телефона.

После отправки формы запись создаётся на стороне бэкенда (`POST /auth/sign-up`). Учётные данные хранятся в зашифрованном виде: пароль не сохраняется в открытом виде, для хеширования используется соль.

Вход выполняется по логину и паролю (`POST /auth/sign-in`). При успешной аутентификации бэкенд возвращает JWT-токен, который фронтенд автоматически прикрепляет к последующим запросам в заголовке `Authorization: Bearer <token>`. Время жизни токена настраивается администратором.

## 3. Главный экран и навигация

После входа открывается главная страница "Загрузка документов". В левой части экрана расположено боковое меню с разделами:

- **Главная** — загрузка документов, превью, запуск анализа.
- **Документы** — файловое хранилище: папки, поиск, переименование, корзина.
- **Аналитика** — агрегированная статистика по проверенным документам и контрагентам.
- **Аккаунт** — личные данные пользователя.
- **Выйти** — завершение сессии.

В правом верхнем углу — поле быстрого поиска по документам и переключатель языка интерфейса. Кнопка "Скрыть меню" в шапке свернёт левую панель до иконок и расширит рабочую область.

## 4. Загрузка документов

На главной странице расположена область **"Загрузите файл(-ы) сюда"**. Способы загрузки:

- перетащить один или несколько файлов из проводника в пунктирную область;
- кликнуть по области и выбрать файлы в стандартном диалоге операционной системы.

После выбора фронтенд отправляет каждый файл на бэкенд через `POST /api/documents` (multipart-форма с полем `file`). Бэкенд:

1. Проверяет MIME-тип и размер.
2. Сохраняет файл в объектное хранилище (S3-совместимое, в стандартной поставке — MinIO).
3. Создаёт запись в таблице `documents` со статусом `queued`.
4. Возвращает идентификатор документа.

Каждый загруженный файл считается отдельной проверкой и отображается в блоке **"Загруженные документы"** под областью загрузки. Кликом по карточке файла открывается превью:

- **PDF** рендерится через `pdf.js` (canvas + текстовый слой, поверх которого работает выделение и подсветка рисков).
- **DOCX** рендерится через `docx-preview` с сохранением заголовков, списков, таблиц и иллюстраций.

## 5. Запуск анализа

После загрузки документ переходит в очередь. Бэкенд автоматически запускает конвейер анализа:

1. Извлечение текста из файла.
2. Обезличивание персональных данных (NER).
3. Запрос к ИИ-модели с инструкцией выделить риски, признаки мошенничества и рекомендации.
4. Сохранение результата в таблицу `analysis_results`.

Статусы документа меняются последовательно: `queued -> processing -> completed`. При сбое статус становится `failed`, поле `error_msg` содержит описание причины.

Фронтенд опрашивает `GET /api/documents/:id/result` и отображает прогресс. Перед запуском анализа открывается модальное окно настроек проверок: чекбоксы, которыми можно отключить ненужные категории (юридические риски, финансовые показатели, формальные требования и т.д.). Снятые галочки уменьшают объём проверяемых тем и ускоряют ответ.

## 6. Работа с результатами

Когда статус документа становится `completed`, фронтенд получает `analysis_results.result_json` и формирует визуальное представление:

- В правой панели появляется список рисков. Каждый риск содержит уровень (Большие / Средние / Малые), название, цитату из документа и комментарий.
- В области превью документа цитаты подсвечиваются цветными метками, соответствующими уровню риска.
- Клик по риску в панели прокручивает превью к нужному фрагменту.
- Клик по подсветке открывает поповер с подробностями и кнопкой **"Принять риск"**. После принятия риск в панели окрашивается зелёным.

Подсветка работает и по PDF, и по DOCX: алгоритм сопоставляет цитату с фактическим текстом документа (нормализация кавычек, тире, переносов строк) и накладывает цветные блоки поверх текстового слоя.

## 7. Чат с ассистентом и экспорт отчёта

Под результатами расположен **чат с ассистентом**. Пользователь может задавать уточняющие вопросы по документу: попросить разъяснить риск, переформулировать пункт, привести аналогичный пример из практики. Контекст разговора привязывается к конкретному документу — переключение на другой файл открывает соответствующую переписку.

Кнопка **"Экспорт отчёта"** формирует итоговый PDF-документ, который содержит:

1. Метаданные документа (имя, размер, дата проверки).
2. Список выявленных рисков с цитатами и комментариями.
3. Сводку рекомендаций.
4. Историю переписки с ассистентом.

Файл сохраняется на устройство пользователя.

## 8. Раздел "Документы"

Раздел открывает файловое хранилище пользователя. Все ранее загруженные документы доступны здесь независимо от того, в какой сессии они были загружены: фронтенд запрашивает `GET /api/documents` и отображает список с учётом фильтра по статусу.

Возможности:

- Создание папок и вложенных структур.
- Поиск по имени файла.
- Сортировка (А-Я / Я-А, по дате изменения).
- Переименование файлов и папок.
- Перемещение в архив и корзину.
- Скачивание исходного файла.

Внизу страницы отображается индикатор использования памяти — текущий объём занятого хранилища относительно лимита тарифа.

## 9. Раздел "Аналитика"

Раздел выводит статистику по проверкам пользователя за выбранный период. Доступные показатели:

- Доля документов, соответствующих стандартам качества (без существенных нарушений).
- Общее количество проверенных документов.
- Количество выявленных рисков и нарушений.
- Количество изученных контрагентов.

Фильтры в верхней части позволяют выбрать период, тип файлов и контрагента. Данные подсчитываются на стороне бэкенда из таблицы `analysis_results` и аудит-журнала.

## 10. Раздел "Аккаунт"

Раздел отображает данные текущего пользователя: ФИО, логин, email, телефон, остаток средств на счёте. Часть полей (имя, фамилия, отчество, телефон) можно отредактировать — фронтенд отправит запрос `PATCH /api/profile`, и бэкенд обновит запись в таблице `user_profiles`.

Логин и email изменению через интерфейс не подлежат.

## 11. Завершение работы

Кнопка **"Выйти"** в боковом меню удаляет JWT-токен из памяти браузера и перенаправляет на форму входа. Сервер хранит только сам факт регистрации пользователя, токен действует до истечения срока жизни — при необходимости администратор может принудительно завершить все сессии в БД (см. инструкцию администратора).
